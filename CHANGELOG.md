# BandWagon v2.0.12 패치노트
(2026-07-10)

## 버그 수정
- **레인 이름을 바꿀 때마다 레인 표가 맨 위로 튕기던 문제** — 이름 변경도
  다른 레인 편집과 같은 커밋 경로(_commit_lanes → _apply_lanes_snapshot)를
  거쳐 표 전체를 다시 그리는데, 그때 스크롤 위치를 저장/복원하지 않아
  레인이 많으면 편집할 때마다 목록이 맨 위로 돌아가 매우 불편했습니다.

## 개선
- **레인 표 기본 높이를 15줄 정도로 확보** — 스크롤 없이 한눈에 더 많은
  레인을 볼 수 있습니다.
- **슬라이더 Alt+드래그 미세조정 속도를 더 느리게(1/8 → 1/80)** — 실사용해보니
  1/8로는 여전히 작은 드래그에 값이 크게(10 안팎) 뛰어, 훨씬 세밀하게
  조정되도록 늦췄습니다.

## 새 기능
- **슬라이더 Alt+드래그로 미세 조정** — 정밀회전/부채꼴(곡률)/기울기(전단)
  보정, 밝기/대비, 경계 임계값/최대 밴드 길이, 합성 스튜디오의 가시광
  투명도까지 모든 슬라이더에서 Alt를 누른 채로 드래그하면 이동량이
  1/8로 줄어 값이 훅훅 뛰지 않고 세밀하게 조정됩니다. Alt 없이 쓰면
  기존과 완전히 동일하게 동작합니다.

## 새 기능
- **가이드 표시 체크박스 추가** — 보정 탭(회전/펴기)의 격자+중앙 십자선
  가이드를 캔버스 위 '가이드 표시' 체크박스로 직접 켜고 끌 수 있습니다.
  꺼두면 보정 탭에 있어도 안 보이고, 다른 탭으로 갔다 와도 상태가
  유지됩니다.

## 새 기능
- **"오버레이만(투명 배경)" 복사/저장 옵션 추가** — 결과 이미지 복사/저장 시
  '사진만'/'분석 포함(합성)'에 더해, 사진 없이 레인 경계·밴드·MW 라벨만
  완전 투명 배경 PNG로 내보내는 옵션이 생겼습니다. 다른 배경 위에 겹쳐
  쓰거나 발표 자료에 붙여넣기 좋습니다.

## 새 기능
- **"파일 > 최근 파일" 추가** — 최근에 연 그림/프로젝트/합성 파일 목록에서
  바로 다시 열 수 있습니다(최대 10개). 지금 창이 비어있으면 그 자리에,
  아니면 새 창에 엽니다("열기"와 동일한 규칙). 파일이 이동/삭제됐으면
  알려주고 목록에서 자동으로 빠집니다.
- **마커 프리셋에 DNA 사다리 3종 추가** — 기존 단백질 11밴드 기본값에
  더해 1kb DNA 사다리(NEB N3232), 100bp DNA 사다리(NEB N3231), 범용
  DNA 마커(λ/HindIII)를 기본 프리셋으로 넣었습니다. 이미 프리셋 파일이
  있는 기존 사용자도 다음 실행 시 이름 대조로 누락된 것만 자동으로
  채워지고, 직접 만든 프리셋은 그대로 보존됩니다.

## 버그 수정
- **캔버스에 파일을 끌어놓으면(drag&drop) 확장자를 무시하고 무조건
  그림으로 열리던 문제** — .bandwagon/.bwcomposite 파일을 끌어놔도
  프로젝트/합성으로 안 열리고 그림 열기를 시도했습니다. 메뉴·파일 연결과
  똑같이 확장자를 보고 분기하도록 고쳤고, "최근 파일" 기록도 이제 함께
  남습니다.

## 버그 수정
- **레인 이름이 MW 라벨과 겹치던 문제** — 마커 맨 위 밴드 값이 250(kDa)

  이미지 위가 아니라 그만큼 캔버스를 위로 늘린 별도 띠에 그려 원본
  이미지와 절대 겹치지 않게 했습니다.

  더 썼습니다. `self._orig` 객체가 실제로 바뀔 때만 다시 해싱하도록
  캐싱해, 사진을 그대로 두고 레인/커브/메모만 만지는 동안은 이 비용이
  거의 0에 가까워졌습니다.

# BandWagon v2.0.2 패치노트
(2026-07-10)

## 버그 수정
- **.bandwagon/.bwcomposite 파일 연결이 실제 설치 후에도 안 되던 문제** —
  v2.0.0에서 `HKCR`을 `HKA`로 바꿔 "액세스 거부" 오류는 해결했지만, `HKA`는

  성공으로 보여도 탐색기는 전혀 인식하지 못했습니다. 모든 `[Registry]`
  Subkey 앞에 `Software\Classes\`를 붙여 실제로 연결되도록 고쳤습니다.

# BandWagon v2.0.0 패치노트
(2026-07-10)

## 새 기능
- **"파일 > 열기"가 새 창을 띄웁니다** — 그림이든 프로젝트든 열면 지금
  창은 그대로 두고 새 창이 하나 더 열려, 여러 젤을 동시에 띄워두고
  비교할 수 있습니다(붙여넣기는 예전처럼 지금 창에 바로 적용).
  웨스턴블롯 창도 각자 자기 메인 창을 부모로 열리므로 창마다 자연히
  구분됩니다.
- **툴바를 메뉴바로 재구성** — 항목이 늘어나면서 한 줄 툴바가 복잡해져,
  파일 / 웨스턴블롯 / 편집 / 정보 / 언어 카테고리 메뉴로 정리했습니다.
  그림 열기·붙여넣기·프로젝트 저장/새로 저장/현재 위치 열기·결과 이미지
  내보내기(복사/저장/CSV)가 전부 "파일" 메뉴 하나에 모여 있고, 웨스턴블롯
  (만들기 / 기존 합성 파일 불러오기)만 성격이 달라 따로 분리했습니다.
- **그림 열기와 프로젝트 열기를 "열기" 하나로 통합** — 파일 확장자를 보고
  그림이면 바로 열고 .bandwagon이면 프로젝트로 불러옵니다.
- **프로젝트 저장이 이제 같은 파일에 덮어씁니다** — 이미 열었거나 저장한
  적 있는 프로젝트라면 Ctrl+S/"프로젝트 저장"이 대화상자 없이 그 자리에
  바로 저장합니다. 새 이름/위치로 저장하려면 "프로젝트 새로 저장"
  (Ctrl+Shift+S)을 쓰면 됩니다. 저장 성공 시 제목표시줄도 그 파일
  이름으로 갱신됩니다.
- **"프로젝트 현재 위치 열기" 추가** — 지금 작업 중인 프로젝트 파일이
  있는 폴더를 탐색기(맥은 Finder)로 바로 엽니다.
- **언어 전환을 명시적 선택으로 변경** — 토글 버튼 대신 "언어" 메뉴에서
  한국어/English를 직접 고릅니다.
- **닫기 전 저장 확인** — 마지막 저장 이후 바뀐 내용이 있는 채로 창을
  닫으려 하면 저장/저장 안 함/취소를 물어봅니다.
- **탐색기에서 .bandwagon/.bwcomposite 파일 연결** — 설치본에서 이제 두
  확장자를 더블클릭하면 BandWagon이 바로 열립니다. .bwcomposite는 아이콘도
  프로젝트 파일과 색만 다르게(청록 계열 -> 자홍 계열) 만들어 탐색기에서
  한눈에 구분됩니다. .bwcomposite를 열면 메인 창에서 바로 분석을 시작하면서,
  빈 합성 스튜디오 창도 비모달로 같이 뜹니다(새로 합성을 만들거나 다른
  합성 파일을 이어서 불러올 수 있게).
- **메모를 위한 새 탭 추가** — 정량 탭 바로 옆에 독립된 "메모" 탭을
  만들어, 분석에 대한 메모를 남길 수 있습니다. 프로젝트 저장/불러오기
  시 함께 저장·복원되고, 새 이미지를 열면 이전 세션의 메모는
  초기화됩니다(레인/보정 등 다른 세션 상태와 동일). (.bandwagon 파일
  스키마가 바뀌어 GELPROJ_FORMAT_VERSION을 2로 올림 — 예전 버전으로 새
  파일을 열면 버전 경고가 뜨지만, 새 버전은 memo 필드가 없는 예전
  파일도 문제없이 엽니다.)

## 버그 수정
- **레인 이름을 편집하는 동안 글자가 안 보이던 문제** — 표에서 이름 칸을
  더블클릭해 편집할 때 뜨는 입력창이 다크 배경에 검정 글씨라 커서가
  가 있는 동안은 아무것도 안 보였습니다. 편집창에도 밝은 글씨색을
  지정했습니다.
- **곡률(부채꼴)/기울기 보정의 가장자리가 밋밋하게 늘어나 보이던 문제**
  — `cv2.remap`의 `BORDER_REPLICATE`(가장자리 픽셀을 그대로 늘여씀) 대신
  `BORDER_REFLECT`(거울처럼 반사)로 바꿔 자연스러워졌습니다. 원본
  픽셀을 실제로 가져오는 방식은 이 시점까지의 회전·자르기 전체를
  누적 계산해야 해서 별도로 검토가 필요합니다.
