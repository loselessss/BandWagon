# BandWagon v1.4.2 패치노트
(2026-07-06)

## 개선
- **설치 언어를 앱 첫 실행 언어로 연동** — 설치 마법사에서 고른 언어가
  실제 앱 언어에는 전혀 반영되지 않던 문제를 고쳤습니다. 이미 언어를
  설정해본 적 있는 재설치/업데이트에서는 기존 설정을 덮어쓰지 않습니다.

## 버그 수정
- **펴기(warp) 실행 후 되돌리기하면 처음 불러온 상태로 완전히 돌아가던
  문제** — 젤 영역을 코너로 지정해 펴기를 실행한 뒤 되돌리기를 하면,
  방금 지정했던 코너가 사라지고 아예 처음부터 코너를 다시 찍어야 했습니다.
  이제 펴기를 되돌리면 그때 썼던 코너 지정 상태로 복귀해서 바로 조정 후
  다시 실행할 수 있습니다.
- **톤 커브 창의 히스토그램 배경선이 거칠게 보이던 문제** — 다른 선들과
  달리 이 선만 둥근 이음매(RoundJoin/RoundCap) 처리가 안 돼 있어 유독
  각지게 보였습니다.

# BandWagon v1.4.1 패치노트
(2026-07-06)

## 버그 수정
- **레인 개수 지정 자동검출 시 레인 폭이 들쭉날쭉하던 문제** — 개수를
  지정해 자동검출할 때 각 경계를 등분선 기준 ±40% 범위 안에서 무조건
  가장 약한 신호 지점(골)으로 스냅했는데, 범위가 넓다 보니 사실상 항상
  뭔가에 끌려가버려 레인 폭이 지그재그로 고르지 않았고, 심하면 폭이 몇
  px밖에 안 되는 레인이 끼어들기도 했습니다(실제 젤 사진으로 확인).
  이제 "레인 폭은 균등하다"를 기본값으로 삼고, 등분선보다 뚜렷하게(75%
  이하) 약한 골이 근처(±15%)에 있을 때만 소폭 보정하도록 바꿔 훨씬
  고르게 나옵니다.

# BandWagon v1.4 패치노트
(2026-07-06)

## 새 기능
- **마커 프리셋 CSV 내보내기/가져오기** — 마커 프리셋 관리 창에
  "내보내기…"/"가져오기…" 버튼을 추가했습니다. 앱 자체 저장소
  (~/.bandwagon_markers.json)와 별개로 CSV 파일로 백업하거나 다른
  컴퓨터/동료와 공유할 수 있습니다. 가져오기는 기존 목록에 추가되는
  방식입니다.

## 버그 수정
- **설치본 작업표시줄/타이틀바 아이콘이 여전히 기본 아이콘으로 뜨던
  문제** — v1.3.1에서 exe 파일 자체에는 아이콘을 심었지만, Qt 창은
  자기 아이콘을 따로 지정해야 해서 실행 중 작업표시줄·타이틀바에는
  여전히 기본 아이콘이 떴습니다. 코드에서 직접 아이콘을 지정하도록
  고쳐서 완전히 해결했습니다.

## 기타
- **About 창의 SHA-256 소스 감사 표시 제거** — 대조할 원본 해시를
  어디에도 공개해두지 않아 실제로는 변조 여부를 검증할 수 없던 죽은
  기능이라 걷어냈습니다.

# BandWagon v1.3.1 패치노트
(2026-07-06)

## 버그 수정
- **부팅 스플래시 화면 테두리 얼룩** — 스플래시 이미지(둥근 모서리 카드)가
  투명 배경(PNG 알파 채널)을 쓰고 있었는데, PyInstaller 네이티브 스플래시가
  반투명 가장자리 픽셀을 제대로 블렌딩하지 않고 그대로 그려서 모서리에
  얼룩진 테두리가 보이던 문제를 고쳤습니다. 이미지를 완전 불투명한
  사각형으로 바꿔 해결했습니다.
- **부팅 스플래시가 로딩 중간에 크기가 바뀌던 문제** — exe에 DPI 인식
  정보가 없어서, Qt가 고해상도(HiDPI) 스케일링을 켜는 순간 Windows가
  스플래시에 적용하던 자동 확대가 갑자기 꺼지며 크기가 툭 바뀌었습니다.
  exe에 Per-Monitor-V2 DPI 인식 매니페스트(bandwagon.manifest)를 심어
  처음부터 일관되게 렌더링되도록 했습니다.
- **설치본 아이콘이 기본(파이썬으로 오해받는) 아이콘으로 나오던 문제** —
  exe에 아이콘을 지정한 적이 없어 작업표시줄·바로가기·설치 마법사에
  전부 PyInstaller 기본 아이콘이 뜨던 문제를 고쳤습니다. 앱 로고 마크로
  만든 bandwagon.ico를 빌드/설치 스크립트에 연결했습니다.

# BandWagon v1.3 패치노트
(2026-07-03)

## 새 기능

### 웨스턴 블롯 합성
- **합성 스튜디오(독립 창)** — 가시광+UV 사진을 코너 4점으로 정렬해
  합성하는 별도 창을 추가했습니다(툴바 "합성 스튜디오"). 합성 결과를
  .bwcomposite 파일 하나로 내보내면, 그 안에 화면용 블렌드와 분석용
  UV 그레이스케일이 함께 저장됩니다.
- **합성 불러오기** — 내보낸 .bwcomposite를 툴바 "합성 불러오기"로 열면
  곧바로 밴드 분석을 시작할 수 있습니다. 합성 과정이 메인 앱의 되돌리기
  히스토리에 원본 사진을 붙들어두지 않아 메모리에 더 가볍습니다.

## 버그 수정 / 내부 정리
- **합성 임포트 후 편집 시 분석 정확도 유지** — 합성 파일을 불러온 뒤
  회전·자르기·펴기 등을 하면 분석용 UV 그레이스케일이 사라져 가시광의
  마커 글자가 다시 밴드로 잡히던 문제를 고쳤습니다. 이제 편집 시 UV
  기준 이미지도 함께 변환되어 정렬이 유지됩니다.
- **구 웨스턴 합성 탭 제거** — 새 합성 스튜디오로 대체된 예전 내장 탭
  코드(western.py)를 걷어내고, 더는 쓰이지 않는 특수 합성 연산과 관련
  문자열을 정리했습니다. 기능·동작 변화는 없습니다.

### 화면 반응성

  캐싱하도록 고쳤습니다. 큰 사진에서 드래그가 훨씬 부드럽습니다.
- **밝기/대비/톤커브 슬라이더 반응 속도** — 드래그 중에는 캔버스
  크기로 축소한 미리보기에만 적용하고, 손을 뗄 때만 원본 해상도로
  다시 계산하도록 바꿨습니다. 결과물(저장/분석용 이미지)에는 영향
  없습니다.

## 새 기능

### 분석
- **세로 분석 범위** — 웰이나 염료 전선(dye front)처럼 정량에서 빼고 싶은
  위/아래 구간을 직접 지정해 제외할 수 있습니다. "레인" 탭에서 범위
  지정 모드를 켜고 이미지 위에서 드래그하면 됩니다.
- **Smear 자동 제외** — 밴드 경계 폭(px)이 일정 값을 넘으면 smear(폭
  넓게 퍼진 신호)로 보고 자동으로 결과에서 뺍니다. "밴드 검출" 그룹의
  "최대 밴드 길이" 슬라이더로 조절합니다(0=꺼짐, 기본값).

## 개선

### 젤 영역 자동 인식
- **조명 불균일에 강해짐** — 사진 한쪽이 더 밝은 경우(촬영장비 흔한
  특성)에도 안정적으로 영역을 잡습니다. 합성 테스트 기준, 기존 방식은
  조명 기울기가 커질수록 인식률이 크게 떨어졌으나 개선 후에는 일정하게
  유지됩니다.
- **마커 레인이 본체와 떨어져 찍힌 경우도 함께 인식** — 마커와 샘플
  레인 사이에 큰 틈이 있어도 마커가 누락되지 않고 같이 잡힙니다.
- 밴드로 인한 내부 얼룩 때문에 인식 영역이 쪼개지던 문제 개선.

## 버그 수정

### 되돌리기(Ctrl+Z)
- 정밀회전·곡률보정 슬라이더를 놓으면 그림은 바뀐 채인데 슬라이더만
  0으로 돌아가던 문제 수정 — 이제 슬라이더가 실제 값을 계속 보여줍니다.
- 밝기·대비·톤커브 조정이 되돌리기에 포함되지 않던 문제 수정.
- 레인 추가/이동/삭제/자동검출/이름변경/종류변경/전체삭제가 되돌리기에
  포함되지 않던 문제 수정. 자동검출처럼 한 번에 여러 레인이 생기는
  작업은 한 번의 되돌리기로 전체가 풀립니다.
- 기록 가능한 단계 수 10 → 20으로 확대.
- 기하변환(회전 등) 이후 되돌리기로 그 이전 레인 구성을 복원해도 곧바로
  다시 비워지던 내부 버그 수정.

### UI 문구
- "코너 4점을 좌상→우상→우하→좌하 순서로 클릭하세요" 등 옛 상호작용
  방식의 안내 문구가 일부 화면(인식 실패 알림, 도움말, 합성 탭)에
  남아있던 것을 현재 방식(드래그로 사각형 생성 후 모서리 조정)에 맞게
  통일.

## 기타
- 웨스턴 블롯 합성 기능을 별도 모듈로 분리.
- 메인 코드(app.py)를 기능별 모듈로 재구성 — 동작에는 영향 없음.
- Windows 설치 마법사 추가(Inno Setup 기반), 관리자 권한 불필요.
- 빌드를 onefile에서 onedir 방식으로 전환 — 실행 시작 속도 개선.
