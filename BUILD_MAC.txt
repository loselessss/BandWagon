[ BandWagon.app 만들기 (macOS) ]

PyInstaller는 크로스 컴파일이 안 됩니다 — Windows/Linux에서는 Mac용
빌드를 만들 수 없고, 반드시 실제 맥에서 이 스크립트를 돌려야 합니다.

1) 터미널에서 이 폴더로 이동 후, 최초 1회만:
     python3 -m pip install pyinstaller

2) 실행권한 부여(처음 한 번만, zip 풀고 나서 권한이 풀릴 수 있음):
     chmod +x build_mac.sh

3) 실행:
     ./build_mac.sh

4) 완료되면 dist/BandWagon.app 생성 — 더블클릭하면 바로 실행됩니다.

윈도우 버전과 다른 점:
  - 스플래시 화면이 안 뜹니다. PyInstaller의 네이티브 스플래시 기능
    자체가 macOS를 지원하지 않습니다(Tcl/Tk 스레드 제약 — PyInstaller
    공식 문서에 명시된 한계). 코드 쪽은 이미 이 경우를 대비해서
    스플래시 닫기 호출을 try/except로 감싸놨으니, 그냥 메인 창이
    바로 뜨는 것뿐이고 따로 손볼 건 없습니다.
  - 인스톨러(installer.iss)는 윈도우(Inno Setup) 전용입니다. 맥은
    보통 .app을 그냥 압축해서 주거나, 원하면 나중에 DMG로 만들 수도
    있습니다(`hdiutil create` — 필요해지면 그때 스크립트 추가하면 됩니다).
  - 앱 아이콘을 넣고 싶으면 .icns 파일이 필요합니다(윈도우의 .ico와
    다른 포맷) — 지금은 이 저장소에 없어서 기본 아이콘으로 빌드됩니다.
    build_mac.sh 안의 --icon 관련 주석을 참고하세요.

버전을 올릴 때는 bandwagon/meta.py의 APP_VERSION만 바꾸면 됩니다
(이 스크립트는 버전 번호를 파일명에 안 박아 넣으므로 따로 맞출 게 없음).
